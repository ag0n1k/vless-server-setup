# legacy — оригинальные shell-скрипты (до Ansible)

Эти каталоги — снимок прежнего состояния:

- `scripts/` — первая итерация (Feb 2025): базовый OVPN + sing-box + TPROXY
- `scripts_v2/` — расширение (Mar 2025): подписки, переключатель нод, health-check, failover

Они оставлены **как документация** того, как стенд был развёрнут изначально.
**Запускать их на боевом хосте 194.87.99.207 не нужно** — конфигурация теперь
управляется через `ansible/` (см. корневой README).

История: ранее `scripts/` лежали в корне репо и применялись вручную через
`bash vpn.sh`, `bash vless.sh`, `bash trpoxy.sh`. Скрипты неидемпотентны
(`gen-dh`, `rm -rf easy-rsa/`, дублирующиеся `ip rule add`), поэтому
повторный запуск ломал состояние. Это и стало причиной перехода на Ansible.

## Что НЕ переехало в Ansible (важно)

На боевом хосте лежат `wg.sh`, `wg-nf.sh`, `wg-proxy.sh`, `wg-client.sh` —
эти скрипты **никогда не были в репо**. Их функционал (WireGuard hub +
NAT + TPROXY-маршрутизация WG-трафика) теперь покрыт ролями
`roles/wireguard/` и `roles/tproxy/`.

`wg-nf.sh` содержит `nft flush ruleset` без guard'а — если случайно
запустить его сейчас, снесёт всю текущую разводку (fail2ban, OpenVPN NAT,
TPROXY). Не запускать ни при каких условиях.
